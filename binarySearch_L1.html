What is a Binary Search?

Binary Search Binary Search: An efficient searching algorithm that finds the position of a target value within a sorted array by repeatedly dividing the search interval in half. is one of the most efficient searching algorithms. It works by repeatedly dividing the search space in half until the target element is found or determined to be absent. This algorithm only works on sorted arrays.

💡 Real-World Analogy: Imagine finding a word in a dictionary 📖 — you open to the middle, check if your word comes before or after, then repeat with the relevant half.

Divide and Conquer Approach

  • Start with the middle element of the sorted array.
  • If the target equals the middle element, you're done.
  • If the target is less than the middle, search the left half.
  • If the target is greater than the middle, search the right half.
  • Repeat until the element is found or the search space is empty.
🧩 Key Principle:
  • Requires a sorted array as prerequisite.
  • Eliminates half of the remaining elements in each step.
  • Much faster than linear search for large datasets: O(log n).

Algorithm: Binary Search (Array A, Value x)

Step 1: Set low to 0 and high to n-1

Step 2: While low ≤ high, repeat the following steps:

Step 3: Set mid = (low + high) / 2

Step 4: If A[mid] == x, return mid

Step 5: If A[mid] < x, set low = mid + 1

Step 6: Else, set high = mid - 1

Step 7: If low > high, return -1

Step 8: Exit

How Binary Search Works

  • Start by comparing the middle element of the array to the target value
  • If the target value is found, return the index of that element
  • If the target value is smaller, narrow the search to the left half of the array
  • If the target value is larger, narrow the search to the right half of the array
  • If the search space is exhausted without finding the target value, return -1

Binary Search Time Complexity

Binary Search has a time complexity of O(log n) where n is the number of elements in the array.

Time Complexity Graph
Time increases logarithmically as number of values increases

⭐ Best Case Scenario: O(1)

The best case occurs when the target element is found at the middle position on the very first comparison.

Time Complexity: O(1) - Constant time

Example:

Array: [1, 3, 5, 7, 9, 11, 13, 15]

Target: 7

Step 1: mid = (0 + 7) / 2 = 3

arr[3] = 7 ✅ Found immediately!

Result: Only 1 comparison needed

⚠️ Worst Case Scenario: O(log n)

The worst case occurs when the target element is at one of the ends or not present in the array, requiring maximum comparisons.

Time Complexity: O(log n) - Logarithmic time

Example:

Array: [1, 3, 5, 7, 9, 11, 13, 15]

Target: 1 (first element)

Step 1: mid = 3, arr[3] = 7 → Search left

Step 2: mid = 1, arr[1] = 3 → Search left

Step 3: mid = 0, arr[0] = 1 ✅ Found!

Result: 3 comparisons needed (log₂8 = 3)

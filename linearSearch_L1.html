What is a Linear Search?

Linear Search Linear Search: A simple searching algorithm that checks every element in the array sequentially until the target is found or the end is reached. is the simplest searching algorithm. It works by sequentially checking each element in the array from start to finish until the target element is found or the array ends. This algorithm works on both sorted and unsorted arrays.

💡 Real-World Analogy: Imagine searching for a specific book on a shelf 📚 — you check each book one by one from left to right until you find the one you're looking for.

Sequential Search Approach

  • Start from the first element of the array.
  • Compare each element with the target value.
  • If a match is found, return the index of that element.
  • If the end of the array is reached without finding the target, return -1.
  • Continue until the element is found or array ends.
🧩 Key Principle:
  • Works on both sorted and unsorted arrays.
  • Checks every element one by one.
  • Simple but can be slow for large datasets: O(n).

How Linear Search Works

  • Start from the first element and move sequentially through the array
  • Compare each element with the target value
  • If a match is found, return the index immediately
  • Continue checking until the target is found or array ends
  • If no match is found after checking all elements, return -1

Linear Search Time Complexity

Linear Search has a time complexity of O(n) where n is the number of elements in the array.

Time Complexity Graph
Time increases linearly as number of values increases

⭐ Best Case Scenario: O(1)

The best case occurs when the target element is the first element in the array.

Time Complexity: O(1) - Constant time

Array: [11, 3, 5, 7, 9, 13, 15, 1]

⚠️ Worst Case Scenario: O(n)

The worst case occurs when the target element is the last element or not present in the array.

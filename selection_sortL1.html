SELECTION SORT- Level-1

What is a Selection Sort?

The Selection Sort algorithm finds the minimum value in an array and moves it to the front of the array.

Algorithm: Selection Sort (Array A)

Step 1: Start from index 0 to n-1

Step 2: Assume the current index as minIndex

Step 3: Check every element after current index to find the smallest

Step 4: If a smaller element is found, update minIndex

Step 5: Swap the current element with the element at minIndex

Step 6: Repeat the process for the rest of the array

Step 7: Array is now sorted

Pseudocode:

        
selectionSort(array, size)
for i from 0 to size - 1 do
set i as the index of the current minimum
for j from i + 1 to size - 1 do
if array[j] < array[current minimum]
set j as the new current minimum index
if current minimum is not i
swap array[i] with array[current minimum]
end selectionSort

Manual Run-Through:

We will sort the array: [64, 25, 12, 22, 11]

Pass 1:
Start at index 0 → Minimum = 11 (minIndex = 4)
Swap 64 and 11 → [11, 25, 12, 22, 64]

Pass 2:
Start at index 1 → Minimum = 12 (minIndex = 2)
Swap 25 and 12 → [11, 12, 25, 22, 64]

Pass 3:
Start at index 2 → Minimum = 22 (minIndex = 3)
Swap 25 and 22 → [11, 12, 22, 25, 64]

Pass 4:
Start at index 3 → The element 25 is already in place.

Result: Sorted array: [11, 12, 22, 25, 64]

How Selection Sort Works:

  • Find the minimum element in the unsorted array.
  • Swap it with the first element.
  • Move the boundary of the unsorted array one element ahead.
  • Repeat until the array is fully sorted.

Selection Sort Implementation:

  • Use two loops: Outer loop for each position, inner loop to find the minimum.
  • Swap only once per outer loop iteration.
  • No extra space is needed (in-place sorting).

Selection Sort Time Complexity:

Selection Sort has a time complexity of O(n²) for all cases.

  • Always compares every pair → n(n-1)/2 comparisons
  • Best, Worst, and Average cases all take O(n²) time

Understanding Merge Sort

💡 What is Merge Sort?

Merge Sort is an efficient, general-purpose sorting algorithm based on the divide-and-conquer paradigm. It works by recursively dividing the array into two halves until each subarray has only one element, then merging those sorted subarrays back together in the correct order. Because a single element is always sorted, the merge step does all the real work.

🤔 Why Use Merge Sort?

  • Guaranteed O(N log N): Unlike Quick Sort, Merge Sort always runs in O(N log N) — even in the worst case.
  • Stable Sort: Equal elements maintain their original relative order — important for complex data.
  • Works on Linked Lists: Ideal for linked list sorting where random access is expensive.

Split a big task list among roommates. Each independently finishes their portion, then results are combined into one completed ordered list.

How it Works & Key Concepts

🔄 The Mechanism

Merge Sort has two phases: the Divide phase and the Merge phase. In the divide phase, the array is split at its midpoint recursively until all subarrays have only one element. In the merge phase, pairs of sorted subarrays are combined into a single larger sorted array by comparing elements from each half and picking the smaller one at each step.

📚 Basic Terminology

Divide: Split the array at the midpoint into two halves recursively.
Conquer: Base case — a single-element array is already sorted.
Merge: Combine two sorted halves into one sorted array using a temp array.
Mid: The middle index calculated as (left + right) / 2 to split the array.

What Happened?

The array was recursively divided into single elements (log N levels), then merged back in sorted order. Each merge level processes all N elements, giving a total time complexity of O(N log N). A temporary array of size N is needed for the merge step, giving O(N) space.

Implementation

1
Top-Down Method (Recursive — Classic)

The classic top-down approach uses recursion to divide the array into halves and a separate merge() helper to combine them. This is the standard implementation taught in most courses and textbooks.

Time Complexity
O(N log N)
Space Complexity
O(N)

Algorithm:

  • mergeSort(arr, left, right): Entry point — divides the array.
  • If left >= right, return (base case — single element).
  • Calculate mid = left + (right - left) / 2.
  • Recursively call mergeSort on left half: [left, mid].
  • Recursively call mergeSort on right half: [mid+1, right].
  • merge(arr, left, mid, right): Combine two sorted halves using a temp array.
C++
2
Bottom-Up Method (Iterative)

The bottom-up approach avoids recursion entirely. It starts by treating each element as a sorted subarray of size 1, then iteratively merges subarrays of increasing sizes (1→2→4→8...) until the whole array is sorted. Same complexity, no call stack overhead.

Time Complexity
O(N log N)
Space Complexity
O(N)

Algorithm:

  • Base: Every single element is a sorted subarray of size 1.
  • Loop with width = 1, 2, 4, 8, ... (doubling each iteration).
  • In each pass, merge adjacent pairs of subarrays of the current width.
  • Calculate left, mid, and right for each pair and call merge().
  • Continue until width >= N — the full array is now sorted.

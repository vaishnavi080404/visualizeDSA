What is a Stack?

Stack Stack: A linear data structure that follows the Last-In-First-Out (LIFO) principle where elements are added and removed from the same end called the "top". is a fundamental linear data structure that operates on the LIFO (Last-In-First-Out) principle. Think of it like a stack of plates — you can only add or remove plates from the top.

💡 Real-World Analogy: Imagine a stack of books 📚 — you can only add a new book on top, and you can only remove the topmost book. The last book you place is the first one you'll remove.

LIFO Principle

  • Last In, First Out — The most recently added element is removed first.
  • All operations happen at the top of the stack.
  • Elements below the top cannot be directly accessed.
  • Used extensively in recursion, expression evaluation, and backtracking.
🧩 Key Characteristics:
  • Dynamic size (can grow or shrink).
  • Restricted access — only top element is accessible.
  • Efficient O(1) operations for push and pop.

Stack Operations

🔼 PUSH Operation

Adds an element to the top of the stack.

push(element):
    if stack is full:
        return "Stack Overflow"
    else:
        top = top + 1
        stack[top] = element

Time Complexity: O(1)

🔽 POP Operation

Removes and returns the top element.

pop():
    if stack is empty:
        return "Stack Underflow"
    else:
        element = stack[top]
        top = top - 1
        return element

Time Complexity: O(1)

👁️ PEEK Operation

Returns the top element without removing it.

peek():
    if stack is empty:
        return "Stack is Empty"
    else:
        return stack[top]

Time Complexity: O(1)

❓ isEmpty Operation

Checks if the stack is empty.

isEmpty():
    if top == -1:
        return true
    else:
        return false

Time Complexity: O(1)

📏 SIZE Operation

Returns the number of elements in the stack.

size():
    return top + 1

Time Complexity: O(1)

How Stack Works

  • Follows LIFO (Last-In-First-Out) principle strictly
  • PUSH adds elements to the top of the stack
  • POP removes and returns the top element
  • PEEK views the top element without removing it
  • Used in function calls, undo operations, expression evaluation

Stack Time Complexity

All stack operations have O(1) constant time complexity, making it highly efficient.

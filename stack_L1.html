Understanding Stack

💡 What is Stack?

A Stack is a linear data structure that stores elements in a specific order called LIFO (Last In, First Out). This means the element that is added last is removed first. You can imagine a stack like a pile of books — you can only place or remove a book from the top.

🤔 Why Use Stack?

Stacks are used when we need to control the order of operations. They are very useful because insertion (push), deletion (pop), and viewing the top element (peek) all happen in constant time. Due to this efficiency, stacks are widely used in programming and system design.

🔄 How Does Stack Work?

A stack works using two main operations: Push (to add an element at the top) and Pop (to remove the top element). Only the top element is accessible at any time. This behavior makes stacks ideal for tasks like function calls, undo/redo actions, expression evaluation, and depth-first search (DFS).

Time Complexity
O(1)
Space Complexity
O(n)

📖

LIFO (Last In First Out) Principle

The LIFO principle means that the last element added to a stack is the first one to be removed:

  • New elements are always pushed on top
  • Removal (pop) also happens only from the top
  • This ensures a strict order: last in → first out

Real-world Examples of LIFO:

  • Stack of plates – The last plate placed on top is the first one you pick up
  • Stack of books – Books are added and removed from the top, so the last book placed is the first one taken
  • Browser back button – The last page you visited is the first one you go back to
  • Undo operation – The last action performed is the first one to be undone
📝

Basic Terminologies

Top
The position of the most recently inserted element. Insertions (push) and deletions (pop) are always performed at the top.
Size
Refers to the current number of elements present in the stack.
Overflow
Condition when trying to push an element onto a full stack.
Underflow
Condition when trying to pop an element from an empty stack.
🔧

Types of Stack

1. Fixed Size Stack

A fixed size stack has a predefined capacity. Once it becomes full, no more elements can be added (this causes overflow). If the stack is empty and we try to remove an element, it causes underflow. Typically implemented using a static array.

Example: Declaring a stack of size 10 using an array

2. Dynamic Size Stack

A dynamic size stack can grow and shrink automatically as needed. If the stack is full, its capacity expands to allow more elements. As elements are removed, memory usage can shrink as well.

Can be implemented using:
  • Linked List → grows/shrinks naturally
  • Dynamic Array (like vector in C++ or ArrayList in Java) → resizes automatically

Note: We generally use dynamic stacks in practice, as they can grow or shrink as needed without overflow issues.

📦 Stack using Array

Fixed Size

A stack implemented using an array maintains an integer array to store elements, a variable capacity for maximum size, and a variable top to track the index of the top element. Initially, top = -1 to indicate an empty stack.

1
Push Operation
▼

Adds an item to the stack. If the stack is full, then it is said to be an Overflow condition.

Time Complexity
O(1)
Space Complexity
O(1)

Algorithm:

  • Check if the stack is full (top == capacity-1)
  • If full, return Stack Overflow error
  • Otherwise, increment top by 1 (top = top + 1)
  • Insert the new value at position top

🔗 Stack using Linked List

Dynamic Size

A stack implemented using a linked list where each element is represented as a node. The head of the linked list acts as the top of the stack. This implementation allows dynamic sizing without overflow.

1
Push Operation
▼

Adds an item to the stack. Unlike array implementation, there is no fixed capacity. Overflow occurs only when memory is exhausted.

Time Complexity
O(1)
Space Complexity
O(1)

Algorithm:

  • Create a new node with the given value
  • Set the new node's next pointer to the current top
  • Update the top pointer to point to this new node

Push Operation (Linked List)

2
Pop Operation
▼

Removes the top item from the stack by moving the top pointer to the next node. If the stack is empty, it results in an Underflow condition.

Time Complexity
O(1)
Space Complexity
O(1)

Algorithm:

  • Check if the stack is empty (top == NULL)
  • If empty, return Stack Underflow error
  • Create a temporary pointer and point it to the current top
  • Update the top pointer to the next node in the list
  • Retrieve the data from the temporary node and free its memory

C++
Java
Python
JavaScript
C 
int pop() {
    // Check if stack is empty
    if (top == NULL) {
        printf("Stack Underflow\n");
        return -1;
    }
    
    Node* temp = top;       // Store current top
    int val = temp->data;   // Get data
    top = top->next;        // Move top to next node
    
    free(temp);             // Release memory
    return val;
}

🔄 Stack using Deque

Hybrid Structure

A Deque (Double-Ended Queue) allows insertion and deletion from both ends. To implement a stack using a deque, we simply restrict operations to only one end (typically the Back or Rear).

Time Complexity
O(1)
Space Complexity
O(n)

💡 Why use Deque?

  • Java: ArrayDeque is faster than the legacy Stack class because it is not synchronized.
  • Python: collections.deque is optimized for fast O(1) additions and pops from both ends.
  • C++: std::deque provides a sequence container that allows fast insertions at both its beginning and its end.

    deque<int> stack;

    stack.push_back(10);

Java (ArrayDeque) 
import java.util.ArrayDeque;
import java.util.Deque;

public class Main {
    public static void main(String[] args) {
        // Deque is an interface; ArrayDeque is the implementation
        Deque<Integer> stack = new ArrayDeque<>();

        // Standard push/pop methods use the front for performance
        stack.push(10);
        stack.push(20);
        stack.push(30);

        System.out.println(stack.pop() + " removed");
        System.out.println("New Top: " + stack.peek());
    }
}

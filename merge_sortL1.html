MERGE SORT- Level-1

What is a Merge Sort?

The Merge Sort algorithm is a divide-and-conquer algorithm that sorts an array by first breaking it down into smaller arrays, and then building the array back together the correct way so that it is sorted.

Algorithm: Steps

Step 1: If length of A > 1, proceed to step 2; otherwise, return.

Step 2: Divide A into two halves: left and right.

Step 3: Recursively call merge sort on the left and right halves.

Step 4: Merge the two halves into a sorted array.

Step 5: Return the sorted array.

Algorithm: Merge Sort (Array A)


MergeSort(A, p, r):
  if p > r 
    return
  q = (p+r)/2
  mergeSort(A, p, q)
  mergeSort(A, q+1, r)
  merge(A, p, q, r)

Manual Run-Through:

We will sort the array [12, 8, 9, 3, 11, 5, 4] using Merge Sort:

Step 1: We start with an unsorted array, and we know that it splits in half until the sub-arrays only consist of one element. The Merge Sort function calls itself two times, once for each half of the array. That means that the first sub-array will split into the smallest pieces first.

[ 12, 8, 9, 3, 11, 5, 4]
[ 12, 8, 9] [ 3, 11, 5, 4]
[ 12] [ 8, 9] [ 3, 11, 5, 4]
[ 12] [ 8] [ 9] [ 3, 11, 5, 4]

Step 2: The splitting of the first sub-array is finished, and now it is time to merge. 8 and 9 are the first two elements to be merged. 8 is the lowest value, so that comes before 9 in the first merged sub-array.

[ 12] [ 8, 9] [ 3, 11, 5, 4]

Step 3: The next sub-arrays to be merged is [ 12] and [ 8, 9]. Values in both arrays are compared from the start. 8 is lower than 12, so 8 comes first, and 9 is also lower than 12.

[ 8, 9, 12] [ 3, 11, 5, 4]

Step 4: Now the second big sub-array is split recursively.

[ 8, 9, 12] [ 3, 11, 5, 4]
[ 8, 9, 12] [ 3, 11] [ 5, 4]
[ 8, 9, 12] [ 3] [ 11] [ 5, 4]

Step 5: 3 and 11 are merged back together in the same order as they are shown because 3 is lower than 11.

[ 8, 9, 12] [ 3, 11] [ 5, 4]

Step 6: Sub-array with values 5 and 4 is split, then merged so that 4 comes before 5.

[ 8, 9, 12] [ 3, 11] [ 5] [ 4]
[ 8, 9, 12] [ 3, 11] [ 4, 5]

Step 7: The two sub-arrays on the right are merged. Comparisons are done to create elements in the new merged array:

3 is lower than 4
Before [ 8, 9, 12] [ 3, 4, 5, 11]
After: [ 3, 8, 9, 12] [ 4, 5, 11]

Step 8: 4 is lower than 8:

Before [ 3, 8, 9, 12] [ 4, 5, 11]
After: [ 3, 4, 8, 9, 12] [ 5, 11]

Step 9: 5 is lower than 8:

Before [ 3, 4, 8, 9, 12] [ 5, 11]
After: [ 3, 4, 5, 8, 9, 12] [ 11]

Step 10: 8 and 9 are lower than 11:

Before [ 3, 4, 5, 8, 9, 12] [ 11]
After: [ 3, 4, 5, 8, 9, 12] [ 11]

Step 11: 11 is lower than 12:

Before [ 3, 4, 5, 8, 9, 12] [ 11]
After: [ 3, 4, 5, 8, 9, 11, 12]

The sorting is finished!

How Merge Sort Works:

  • Merge Sort is a divide and conquer algorithm.
  • The array is recursively divided into two halves until each subarray has only one element.
  • Then, the subarrays are merged back together in sorted order.
  • It guarantees a sorted array in O(n log n) time complexity.

Merge Sort Implementation:

  • Split the array into two halves.
  • Recursively apply merge sort on each half.
  • Merge the sorted halves to produce the final sorted array.

Merge Sort Time Complexity:

Merge Sort has a time complexity of O(n log n), where n is the number of elements in the array.

  • In the best, worst, and average cases, the time complexity remains O(n log n).
  • Unlike other algorithms like bubble sort, it performs efficiently on large arrays.

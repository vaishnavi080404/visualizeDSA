Quick Sort - Basics

A fast divide-and-conquer sorting algorithm that partitions around a pivot element.

Understanding Quick Sort

💡 What is Quick Sort?

Quick Sort is a highly efficient divide-and-conquer sorting algorithm. It works by selecting a pivot element and partitioning the array into two subarrays — elements less than the pivot on the left, and elements greater than the pivot on the right. The pivot is then in its correct final position, and the process repeats recursively on both halves.

🤔 Why Use Quick Sort?

  • Very Fast in Practice: Average-case O(N log N) with small constants — often faster than Merge Sort.
  • In-Place Sorting: Requires only O(log N) stack space — no extra array needed unlike Merge Sort.
  • Cache Friendly: Accesses memory sequentially during partitioning — great for CPU cache performance.

Real-World Analogy

Think of quick sort as picking a reference point and grouping everything around it, then repeating — just like these everyday situations:

1
Organising People by Age at a Party
Pick one person as the "pivot". Everyone younger moves to their left, everyone older to their right. Repeat within each group independently.
2
Sorting Clothes by Size
Pick a medium-sized shirt as the reference. Put all smaller shirts in one pile, larger in another. Then sort each pile the same way.
3
Seating Guests by Ticket Number
Pick a middle ticket number. All lower numbers go left of the hall, higher numbers go right. Then each side is partitioned again until everyone is seated.
4
Grading Papers Around a Median Score
Pick the median score as a pivot. Papers below go in one stack, above in another. Each stack is partitioned further until all papers are in order.

How it Works & Key Concepts

🔄 The Mechanism

Quick Sort picks a pivot element (commonly the last element), then runs a partition operation that rearranges the array so all elements smaller than the pivot come before it and all larger elements come after it. After partitioning, the pivot is in its final correct position. The algorithm then recursively sorts the left and right subarrays independently.

📚 Basic Terminology

Pivot: The chosen element that partitions the array into two halves.
Partition: The process of rearranging elements around the pivot into left (< pivot) and right (> pivot).
i (pointer): Tracks the boundary of elements smaller than the pivot during partitioning.
In-Place: Sorting is done within the original array — no temporary array needed.

What Happened?

In each recursive call, a pivot was chosen and placed in its final sorted position via partitioning. The left and right subarrays were then sorted independently. On average this takes O(N log N) time. The worst case O(N²) occurs when the pivot is always the smallest or largest element (e.g. already-sorted arrays with last-element pivot), which can be mitigated by random pivot selection.

Implementation

1
Lomuto Partition Scheme (Last Element as Pivot)

The Lomuto scheme picks the last element as the pivot. It uses a single index i to track the boundary of the smaller-elements region. After scanning, the pivot is swapped into its correct position at index i+1.

Time (Average)
O(N log N)
Time (Worst)
O(N²)
Space Complexity
O(log N)

Algorithm:

  • Choose pivot = arr[high] (last element).
  • Initialize i = low - 1 (boundary of smaller elements).
  • Loop j from low to high - 1.
  • If arr[j] ≤ pivot: increment i, swap arr[i] with arr[j].
  • After loop, swap arr[i+1] with arr[high] — pivot is now in correct position.
  • Recursively call quickSort on [low, pivotIndex-1] and [pivotIndex+1, high].

2
Hoare Partition Scheme (First Element as Pivot)

The Hoare scheme (original Quick Sort design) picks the first element as the pivot. It uses two pointers — one starting from the left and one from the right — moving towards each other and swapping elements when they are on the wrong side. Generally performs fewer swaps than the Lomuto scheme.

Time (Average)
O(N log N)
Time (Worst)
O(N²)
Space Complexity
O(log N)

Algorithm:

  • Choose pivot = arr[low] (first element).
  • Initialize i = low - 1 and j = high + 1 (two pointers).
  • Move i right until arr[i] >= pivot. Move j left until arr[j] <= pivot.
  • If i < j, swap arr[i] and arr[j].
  • When i >= j, j is the partition index — recurse on [low, j] and [j+1, high].
  • Note: Hoare does NOT place pivot in its final position directly — both halves must include the partition index.

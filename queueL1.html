What is a Queue?

Queue Queue: A linear data structure that follows the First-In-First-Out (FIFO) principle where elements are added at one end (Rear) and removed from the other (Front). is a fundamental data structure operating on the FIFO (First-In-First-Out) principle. It resembles a real-world line.

💡 Real-World Analogy: A movie ticket line 🍿 — the person who arrives first is the first to be served and leave the line.

FIFO Principle

  • First In, First Out — The earliest element added is the first to be removed.
  • Insertion happens at the Rear.
  • Deletion happens at the Front.
  • Used in CPU scheduling and asynchronous data transfer.
🧩 Key Characteristics:
  • Two open ends: Front and Rear.
  • Order-preserving structure.
  • O(1) time complexity for Enqueue and Dequeue.

Queue Operations

➕ ENQUEUE Operation

Adds an element to the rear of the queue.

enqueue(element):
    if queue is full:
        return "Queue Overflow"
    else:
        rear = rear + 1
        queue[rear] = element

Complexity: O(1)

➖ DEQUEUE Operation

Removes an element from the front of the queue.

dequeue():
    if queue is empty:
        return "Queue Underflow"
    else:
        element = queue[front]
        front = front + 1
        return element

Complexity: O(1)

👁️ FRONT/PEEK Operation

Gets the value of the front element.

front():
    if queue is empty:
        return "Queue is Empty"
    else:
        return queue[front]

Complexity: O(1)

❓ isEmpty Operation

Checks if the queue is empty.

isEmpty():
    if front > rear:
        return true
    else:
        return false

Complexity: O(1)

How Queue Works

  • Follows FIFO (First-In-First-Out) principle.
  • ENQUEUE: Elements join the line at the REAR.
  • DEQUEUE: Elements leave the line from the FRONT.
  • Used in Print Buffers and Task Scheduling.

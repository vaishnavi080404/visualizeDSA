QUEUE- Level-1

What is a Queue?

A Queue is a linear data structure that follows the First In First Out (FIFO) principle. This means that the element inserted first is the one to be removed first, just like a queue of people at a ticket counter.

Basic Queue Operations:

  • Enqueue: Add an element to the end of the queue.
  • Dequeue: Remove the element from the front of the queue.
  • Front: Get the front item from the queue.
  • Rear: Get the last item from the queue.
  • isEmpty: Check if the queue is empty.
  • isFull: Check if the queue is full (in case of fixed-size implementation).

Real-Life Example:

Think of a queue at a movie theater:

  • The first person to enter the queue is the first one to buy a ticket and leave (FIFO).
  • New people can only join from the rear.
  • Tickets are given to the person at the front of the queue first.

Applications of Queue:

  • CPU Scheduling
  • Handling of requests in a web server
  • Breadth-First Search (BFS) in Graphs
  • Print Queue Management
  • IO Buffers, pipes, file IO etc.

Queue Implementation Methods:

  • Using Arrays (Static Queue)
  • Using Linked List (Dynamic Queue)
  • Circular Queue (Overcomes limitations of regular array-based queues)
  • Double-Ended Queue (Deque)
  • Priority Queue

Time Complexity:

  • Enqueue: O(1)
  • Dequeue: O(1)
  • Front/Rear: O(1)

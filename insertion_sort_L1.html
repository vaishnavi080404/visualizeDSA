Insertion Sort - Basics

A simple sorting algorithm that builds the sorted array one element at a time by inserting each into its correct position.

Understanding Insertion Sort

💡 What is Insertion Sort?

Insertion Sort is a simple comparison-based sorting algorithm that builds the final sorted array one element at a time. It works similarly to how you sort a hand of playing cards — pick a card, and insert it into the correct position among the already-sorted cards in your hand. Elements are shifted right to make space for each new insertion.

🤔 Why Use Insertion Sort?

  • Efficient for Small / Nearly Sorted Data: Best case O(N) — extremely fast when the input is already mostly sorted.
  • Stable Sort: Equal elements keep their original relative order — safe for multi-key sorting.
  • Online Algorithm: Can sort a list as it receives elements one at a time — no need for the full input upfront.
Time Complexity (Worst)
O(N²)
Time Complexity (Best)
O(N)
Space Complexity
O(1)

Real-World Analogy

Think of insertion sort as picking up one item at a time and sliding it into the right spot among what you've already organised — just like these everyday situations:

1
Sorting a Hand of Playing Cards
You pick up cards one by one. Each new card is slid left past larger cards until it reaches its correct position in your already-sorted hand.
2
Filing Documents into a Sorted Folder
As new invoices arrive daily, you slide each one past the others in the folder until it sits in the correct date-order position.
3
Inserting a New Student into a Ranked List
A new student joins mid-term. Their score is compared against the sorted leaderboard from the bottom up, shifting names down until the right slot is found.
4
Arranging Books Alphabetically as You Unpack
Each book taken from the box is walked along the shelf, shifting books right, until it reaches the correct alphabetical position — one at a time.

How it Works & Key Concepts

🔄 The Mechanism

Insertion Sort divides the array into a sorted portion (left) and an unsorted portion (right). In each pass, it picks the first element of the unsorted portion as the key, then scans backwards through the sorted portion — shifting elements one position right as long as they are greater than the key. The key is then dropped into the resulting gap.

📚 Basic Terminology

Key: The current element being picked up and placed into its correct position in the sorted portion.
Sorted Portion: The left part of the array that is always kept in sorted order after each pass.
Shift: Moving an element one position to the right to create a gap for the key to be inserted.
Online Algorithm: Can process elements one at a time as they arrive — does not need the full array up front.

What Happened?

In each of the N-1 passes, the next unsorted element was picked as the key and shifted into its correct position within the sorted portion. Elements larger than the key were shifted one position right to make room. If the array were already sorted, no shifts would occur at all — giving best-case O(N). Total comparisons in the worst case: N*(N-1)/2.

Implementation

1
Iterative Method (Standard Loop)

The standard iterative approach uses an outer loop to pick each key (from index 1 to N-1), and an inner while-loop that shifts elements rightward until the correct insertion position is found.

Time Complexity
O(N²)
Best Case
O(N)
Space Complexity
O(1)

Algorithm:

  • Loop i from 1 to N-1 (outer loop — pick each key).
  • Store arr[i] as the key and set j = i - 1.
  • While j >= 0 and arr[j] > key: shift arr[j] to arr[j+1], decrement j.
  • Place the key at arr[j+1] — its correct sorted position.
  • Repeat until all elements are processed.
C++
2
Recursive Method

The recursive approach first sorts the first N-1 elements, then inserts the last element (arr[n-1]) into its correct position within the now-sorted subarray. The key insight is that insertion of one element into a sorted array is the base operation.

Time Complexity
O(N²)
Space Complexity
O(N)

Algorithm:

  • Base Case: If n <= 1, array is already sorted — return.
  • Recursive Step: Call insertionSort(arr, n-1) to sort the first n-1 elements.
  • Store the last element arr[n-1] as the key.
  • Shift all elements in the sorted portion that are > key one position right.
  • Place the key in the resulting gap at its correct position.

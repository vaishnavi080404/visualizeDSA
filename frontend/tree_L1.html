Understanding Tree

💡 What is a Tree?

A Tree is a hierarchical data structure used to organize and represent data in a parent–child relationship. It consists of nodes, where the topmost node is called the root, and every other node can have one or more child nodes. Unlike arrays or linked lists, data in a tree is not stored sequentially — it is organized across multiple levels.

🤔 Why Use a Tree?

Trees are useful for storing data that naturally forms a hierarchy. File systems, the HTML DOM, databases (B-Trees), and network routing all rely on trees. They enable efficient data organization, retrieval, and hierarchical representation.

🔄 Why is Tree a Non-Linear Data Structure?

Data in a tree is not stored sequentially. Instead, it is organized across multiple levels forming a hierarchical structure. Because of this, a tree is classified as a non-linear data structure.

Tree Structure and Terminologies

A tree is hierarchical — each node can have multiple children but only one parent. The root has no parent.

📝

Basic Terminologies

Understanding the vocabulary of trees is essential before working with them:

Root Node
The topmost node with no parent. Example: 15 is the root.
Parent Node
An immediate predecessor of another node. Example: 35 is parent of 3 and 6.
Child Node
An immediate successor of another node. Example: 3 and 6 are children of 35.
Leaf Node
Nodes with no children (External Nodes). Example: 1, 10, 12, 9 are leaf nodes.
Ancestor
Any node on the path from root to a given node. Example: 15 and 35 are ancestors of 10.
Descendant
Node x is descendant of y if y is ancestor of x. Example: 1, 10, 6 are descendants of 35.
Sibling
Nodes sharing the same parent. Example: 1 and 10 are siblings.
Internal Node
A node with at least one child. All non-leaf nodes are internal nodes.
Subtree
A node and all its descendants together form a subtree.
Level of a Node
Number of edges from root to that node. Root is at level 0.
Height of Tree
Length of the longest path from root to any leaf node.
Degree of a Node
Number of children a node has. A leaf node has degree 0.
🌍

Real-World Analogy

Trees appear everywhere in the real world. Here are some familiar examples:

1
File System
Folders contain subfolders and files. The root drive (C:/) is the root; nested folders are children.
2
HTML DOM
The <html> tag is the root. <head> and <body> are children with their own child nodes.
3
Family Tree
Represents ancestor–descendant relationships. Grandparents are ancestors of grandchildren.
4
Organization Chart
CEO is the root. Managers are children, and their employees are leaf nodes of the org tree.
📊

Properties of Tree

  • Number of edges: A tree with N nodes always has exactly N − 1 edges. There is exactly one path between any two nodes.
  • Depth of a node: The number of edges from the root to that node. The root has depth 0.
  • Height of the tree: Length of the longest path from root to any leaf node.
  • Degree of a node: Number of children it has. Degree of tree = max degree among all nodes.
  • Leaf node degree: A leaf node always has degree 0 — no children.
🔧

Types of Tree Data Structures

Tree data structures are classified based on the number of children each node can have:

1. Binary Tree

Each node can have a maximum of two children (left and right).

Full Binary Tree
Every node has either 0 or 2 children. No node has exactly 1 child.
Complete Binary Tree
All levels fully filled except possibly the last, filled left to right.
Balanced Binary Tree
Height difference between left and right subtrees of every node is at most 1.

2. Ternary Tree

Each node can have at most three children, often labeled as left, middle, and right.

3. N-ary Tree (Generic Tree)

Each node can have any number of children. Each node contains data and a list of references to its children (duplicates not allowed).

  • Unlike linked lists, nodes store references to multiple child nodes
  • A binary tree is a special case of N-ary tree where N = 2
  • Used in file systems, DOM trees, and organizational hierarchies

📦 Full Tree Implementation (C++)

Complete Code
C++ — Full with Output 
// Note: Ensure the index is within bounds to avoid overflow errors
#include <iostream>
#include <vector>
using namespace std;

class Node {
public:
    int data;
    vector<Node*> children;
    Node(int x) { data = x; }
};

void addChild(Node* p, Node* c) { p->children.push_back(c); }

void printParents(Node* node, Node* parent) {
    cout << node->data << " -> "
         << (parent ? to_string(parent->data) : "NULL") << "\n";
    for (auto c : node->children) printParents(c, node);
}

void printLeafNodes(Node* node) {
    if (node->children.empty()) { cout << node->data << " "; return; }
    for (auto c : node->children) printLeafNodes(c);
}

void printDegrees(Node* node, Node* parent) {
    int d = node->children.size(); if (parent) d++;
    cout << node->data << " -> " << d << "\n";
    for (auto c : node->children) printDegrees(c, node);
}

int main() {
    Node* root = new Node(1);
    Node* n2 = new Node(2), *n3 = new Node(3);
    Node* n4 = new Node(4), *n5 = new Node(5);
    addChild(root,n2); addChild(root,n3);
    addChild(n2,n4);   addChild(n2,n5);

    cout << "Parents:\n";     printParents(root, nullptr);
    cout << "\nLeaf nodes: "; printLeafNodes(root);
    cout << "\n\nDegrees:\n"; printDegrees(root, nullptr);
    return 0;
}
/* Output:
Parents:
1 -> NULL  2 -> 1  4 -> 2  5 -> 2  3 -> 1
Leaf nodes: 4 5 3
Degrees:
1->2  2->3  4->1  5->1  3->1 */

Importance of Tree Data Structure

  • Trees are useful for storing data that naturally forms a hierarchy
  • File systems are structured as trees — folders containing subfolders and files
  • The DOM of an HTML page is a tree: <html> is the root, <head> and <body> are children
  • Trees allow efficient organization and retrieval of hierarchical data
  • Used in databases (B-Trees), compilers (parse trees), and AI (decision trees)
  • Binary Search Trees enable average O(log n) search, insert, and delete operations
  • Heap trees power priority queues used in Dijkstra's and Prim's algorithms

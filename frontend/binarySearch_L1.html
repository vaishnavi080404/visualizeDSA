Understanding Binary Search

💡 What is Binary Search?

Binary search is an efficient searching algorithm that works on sorted arrays. It repeatedly divides the search interval in half, eliminating half of the remaining elements in each step. This divide-and-conquer approach makes it significantly faster than linear search for large datasets.

🤔 Why Use Binary Search?

  • Logarithmic Time: Much faster than linear search - O(log N) vs O(N).
  • Scalability: Performs exceptionally well on large sorted datasets.
  • Divide and Conquer: Teaches fundamental algorithmic thinking patterns.
Time Complexity (Worst)

Real-World Analogy

Think of binary search as the smart way to find something when the data is already sorted — you always eliminate half the possibilities with each guess:

1
Guessing a Number Game
Someone picks a number 1–100. You guess 50. They say "lower" — now you guess 25. Each guess cuts the range in half.
2
Finding a Word in a Dictionary
You open the dictionary in the middle. If your word comes later alphabetically, you ignore the first half entirely and repeat.
3
Opening a Book to a Page
To reach page 300 in a 400-page book, you open near the middle, see page 200, then flip to the right half — done in a few moves.
4
Finding a Contact in a Sorted List
Contacts sorted A–Z: jump to "M", the name starts with "T" so skip the first half, then narrow down within T's quickly.

How it Works & Key Concepts

🔄 The Mechanism

Binary search works by maintaining three pointers: low, high, and mid. It calculates the middle element and compares it with the target. Based on the comparison:

  • If target == mid: Element found! Return the index.
  • If target > mid: Search in the right half (low = mid + 1).
  • If target < mid: Search in the left half (high = mid - 1).

This process repeats until the element is found or low > high (element doesn't exist).

📚 Basic Terminology

Low: Starting index of the current search space.
High: Ending index of the current search space.
Mid: Middle index calculated as (low + high) / 2.
Sorted Array: Binary search ONLY works on sorted data.

Implementation

1
Iterative Method (Standard Loop)

The iterative approach uses a while loop to repeatedly narrow down the search space by adjusting the low and high pointers based on comparisons with the middle element.

Time Complexity
O(log N)
Space Complexity
O(1)

Algorithm:

  • Initialize low = 0 and high = n - 1.
  • While low <= high:
  •    Calculate mid = (low + high) / 2.
  •    If arr[mid] == target, return mid.
  •    If arr[mid] < target, set low = mid + 1 (search right).
  •    If arr[mid] > target, set high = mid - 1 (search left).
  • If loop exits, return -1 (not found).
C++
2
Recursive Method

The recursive approach uses function calls to divide the problem. Each recursive call works on a smaller portion of the array until the target is found or the search space is exhausted.

Time Complexity
O(log N)
Space Complexity
O(log N)

Algorithm:

  • Base Case 1: If low > high, return -1 (not found).
  • Calculate mid = (low + high) / 2.
  • Base Case 2: If arr[mid] == target, return mid.
  • If arr[mid] < target, recursively search the right half.
  • If arr[mid] > target, recursively search the left half.

Understanding Queue

💡 What is Queue?

A Queue is a linear data structure that stores elements in a specific order called FIFO (First In, First Out). This means the element that is added first is removed first. You can imagine a queue like a line of people waiting to purchase tickets — the first person in line is the first person served.

🤔 Why Use Queue?

Queues are used when we need to maintain order and process items in the sequence they arrive. They are very useful because insertion (enqueue), deletion (dequeue), and viewing the front element (peek) can all happen efficiently. Due to this efficiency, queues are widely used in task scheduling, breadth-first search (BFS), and handling asynchronous requests.

🔄 How Does Queue Work?

A queue works using two main operations: Enqueue (to add an element at the rear) and Dequeue (to remove the front element). Only the front element is accessible for removal at any time. This behavior makes queues ideal for tasks like printer job scheduling, call center systems, CPU task scheduling, and breadth-first search (BFS).

📖

FIFO (First In First Out) Principle

The FIFO principle means that the first element added to a queue is the first one to be removed:

  • New elements are always added at the rear
  • Removal (dequeue) always happens from the front
  • This ensures a strict order: first in → first out

Real-world Examples of FIFO:

  • Line at a restaurant – The first customer to arrive is the first one seated
  • Checkout lane – Customers are served in the order they join the line
  • Traffic at a toll booth – Vehicles pass through in the order they arrive
  • Boarding a bus – Passengers board in the order they arrive at the bus stop
📝

Basic Terminologies

Front
Position of the entry in a queue ready to be served. The first entry that will be removed from the queue.
Rear
Position of the last entry in the queue, that is, the one most recently added. Also called the tail of the queue.
Size
Refers to the current number of elements present in the queue.
Capacity
Refers to the maximum number of elements the queue can hold.
Overflow
Condition when trying to add an element to a full queue.
Underflow
Condition when trying to remove an element from an empty queue.

Types of Queue

1. Simple Queue (Linear Queue)

A simple queue follows the FIFO principle strictly. Insertion happens at the rear and deletion at the front. Can be implemented using arrays or linked lists. In array implementation, there may be wasted space as elements are dequeued.

Note: In simple array implementation, dequeue can be O(n) due to shifting elements, or space is wasted.

2. Circular Queue

A circular queue is an efficient array implementation where the last position is connected back to the first position, forming a circle. This overcomes the space wastage problem of simple queues.

Advantage: Both enqueue and dequeue operations are O(1), and there is no wasted space.

3. Double-Ended Queue (Deque)

In a deque, insertion and deletion can be performed from both ends (front and rear). This provides more flexibility than a simple queue.

Operations:
  • Insert at front or rear
  • Delete from front or rear

4. Priority Queue

A queue where each element is assigned a priority, and deletion always happens based on priority (not just position). The element with the highest priority is removed first.

Use cases: CPU scheduling, Dijkstra's algorithm, Huffman coding

🔄 Circular Queue using Array

Efficient Implementation

A circular queue overcomes the space wastage problem of simple array queues. The rear position wraps around to the beginning when it reaches the end, forming a circle. This allows O(1) enqueue and dequeue operations.

Time Complexity (All Operations)
O(1)
Space Complexity
O(n)

Adds an item at the rear using circular indexing: rear = (front + size) % capacity

O(1)
Space Complexity
O(1)

Algorithm:

  • Check if the queue is full (size == capacity)
  • If full, return Queue Overflow error
  • Calculate rear = (front + size) % capacity
  • Insert element at rear position
  • Increment size by 1

🔗 Queue using Linked List

Dynamic Size

A queue implemented using a linked list where each element is represented as a node. We maintain both front and rear pointers. This implementation allows dynamic sizing without overflow.

1
Enqueue Operation
▼

Adds an item to the rear of the queue. Unlike array implementation, there is no fixed capacity. Overflow occurs only when memory is exhausted.

Time Complexity
O(1)
Space Complexity
O(1)

Algorithm:

  • Create a new node with the given value
  • If the queue is empty, set both front and rear to this new node
  • Otherwise, link the current rear's next to this new node
  • Update the rear pointer to point to this new node

Java
Python
JavaScript
C 
void enqueue(int x) {
    // Create a new node
    Node* temp = (Node*)malloc(sizeof(Node));
    temp->data = x;
    temp->next = NULL;
    
    // If queue is empty
    if (front == NULL) {
        front = rear = temp;
    } else {
        // Add at the rear
        rear->next = temp;
        rear = temp;
    }
}

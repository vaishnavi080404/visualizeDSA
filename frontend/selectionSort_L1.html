Selection Sort - Basics

A simple comparison-based sorting algorithm that selects the minimum element repeatedly.

Level 1 of 3 | Basics

Understanding Selection Sort

💡 What is Selection Sort?

Selection Sort is a simple comparison-based sorting algorithm. It works by dividing the array into two parts — a sorted portion and an unsorted portion. In each pass, it selects the minimum element from the unsorted portion and places it at the beginning of the unsorted portion, growing the sorted side one element at a time.

🤔 Why Use Selection Sort?

  • Simple to Understand: Very easy to implement and great for learning sorting fundamentals.
  • Minimal Swaps: At most N-1 swaps — useful when swap cost is high.
  • In-Place Sorting: Requires O(1) extra memory — no additional array needed.
Time Complexity (Worst)
O(N²)
Time Complexity (Best)
O(N²)
Space Complexity
O(1)

How it Works & Key Concepts

🔄 The Mechanism

Selection Sort maintains two subarrays: a sorted subarray at the beginning and an unsorted subarray for the rest. In each iteration, it finds the minimum element from the unsorted subarray and swaps it with the first element of the unsorted part. This process repeats until the entire array is sorted.

📚 Basic Terminology

Sorted Subarray: The left portion of the array that is already in sorted order.
Unsorted Subarray: The right portion still to be sorted in future passes.
Minimum Index: Index of the smallest element found in the current unsorted portion.
In-Place: Sorting is done within the original array with O(1) extra space.

What Happened?

In each of the N-1 passes, the minimum of the unsorted portion was found and swapped into its correct position. The sorted portion grew by one element per pass until the entire array was sorted. Total comparisons: N*(N-1)/2.

Implementation

1
Standard Method (Nested Loop)

The standard approach uses two nested loops. The outer loop moves the boundary of the sorted subarray one element at a time. The inner loop scans the remaining unsorted portion to find the minimum element.

Time Complexity
O(N²)
Space Complexity
O(1)

Algorithm:

  • Loop i from 0 to N-1 (outer loop — position to fill).
  • Assume arr[i] is the minimum; store its index as min_idx.
  • Loop j from i+1 to N-1 (inner loop — scan unsorted part).
  • If arr[j] < arr[min_idx], update min_idx = j.
  • After inner loop, swap arr[min_idx] with arr[i].
  • Repeat until the full array is sorted.

2
Recursive Method

The recursive approach applies the same selection sort logic by breaking it into smaller subproblems. Each call places the minimum of the remaining subarray into its correct position, then recurses on the rest.

Time Complexity
O(N²)
Space Complexity
O(N)

Algorithm:

  • Base Case: If start index >= N-1, array is sorted — return.
  • Find the index of the minimum element from start to N-1.
  • Swap arr[start] with arr[min_idx].
  • Recursive Step: Call the function with start + 1 to sort the remaining subarray.

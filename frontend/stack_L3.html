Time & Space Complexity

All core stack operations run in constant time — that's what makes it so powerful.

Operation Time Complexity Space Notes
push(val)O(1)O(1)Add to top — no shifting needed
pop()O(1)O(1)Remove from top — direct access
peek() / top()O(1)O(1)Read top without removing
isEmpty()O(1)O(1)Check if stack has elements

Advantages & Disadvantages

Every data structure is a tradeoff. Know when stack shines — and when it doesn't.

✅ Advantages
✓

O(1) push and pop

All core operations are constant time — no looping, no shifting, no searching.

✓

Simple mental model

LIFO is intuitive — like a stack of plates. Easy to reason about, easy to implement.

✓

Perfect for reversal

Push everything in, pop everything out — you get the reverse. No extra code needed.

✓

Recursion management

Every recursive function implicitly uses the call stack. Explicit stacks let you convert recursion to iteration.

✓

Minimal memory overhead

No pointers between nodes (with array implementation). Cache-friendly and memory-efficient.

❌ Disadvantages
✗

No random access

You can only access the top element. Reaching the 5th item means popping 4 items first.

✗

Stack overflow risk

Too many pushes (especially in recursion) can exceed memory limits and crash your program.

✗

No searching

Finding a specific value requires O(n) time — you must pop everything to find something in the middle.

✗

Fixed order only

You're locked into LIFO. If you need FIFO (first-in, first-out), you need a queue instead.

✗

Not suitable for sorting

Sorting a stack requires popping all elements, sorting them, then pushing back — very inefficient.

🎯
When to Use Stack — and When to Avoid It

Choosing the right data structure is half the interview. Here's the decision framework.

✅ Use Stack When...
🔢

You need LIFO order

Last operation must be undone first. Undo/Redo, back navigation, function calls — all LIFO.

🔤

Matching / nesting problems

Brackets, parentheses, HTML tags, XML. Opening tags push, closing tags pop and verify. ( [ { } ] )

↔️

Reversing a sequence

Push all elements, then pop — instant reversal. Great for reversing words, sentences, linked lists.

📈

Next Greater / Smaller Element

Monotonic stack pattern. Find next warmer day, next taller building — O(n) using stack instead of O(n²) brute force.

🌀

Converting recursion to iteration

DFS, tree traversal, pathfinding — replace the call stack with an explicit stack to avoid stack overflow.

🧮

Expression evaluation

Parsing math expressions, postfix/prefix notation, spreadsheet formulas — all use stacks.

❌ Avoid Stack When...
🔍

You need to search for an element

Use a HashMap or Set instead. Searching a stack is O(n) and destroys the stack in the process.

🎲

You need random access

Use an Array or ArrayList. If you need the 3rd or 10th element, a stack forces you to pop everything above it.

🚶

You need FIFO order

Use a Queue. If the first thing added should be the first thing removed (like a task queue or BFS), stack is the wrong tool.

📊

You need to sort data

Use Arrays with built-in sort. Sorting a stack requires O(n²) operations — completely inefficient.

🔗

You need bidirectional traversal

Use a Deque (double-ended queue). If you need to access both ends efficiently, stack only gives you one end.

♾️

Deep recursion without limit checks

Add iterative alternative or increase stack size. Unlimited recursion = stack overflow. Always add a depth limit.

📋 PSEUDO CODE
Factorial
BASE: if n == 0, return 1
CALL: return n × factorial(n-1)
RETURN: unwind, multiply back
Fibonacci
BASE: if n ≤ 1, return n
LEFT: fib(n-1) branch
RIGHT: fib(n-2) branch
RETURN: left + right
Sum of N / Power
BASE: if n == 0, return 0
CALL: return n + sum(n-1)
BASE: if exp == 0, return 1
CALL: return b × power(b, exp-1)
Countdown / Print Digits
BASE: if n < 0, return
CALL: print n, countdown(n-1)
BASE: if n == 0, return
CALL: printDigits(n/10), print n%10
Reverse String / Palindrome
BASE: if len ≤ 1, return s
CALL: reverse(s[1:]) + s[0]
BASE: if len ≤ 1, return true
CALL: s[0]==s[-1] && isPal(s[1:-1])
Binary Search / GCD
BASE: lo > hi → -1
CALL: split at mid, recurse half
BASE: if b == 0, return a
CALL: return gcd(b, a % b)

Best Case — O(1) Base Case Hit Immediately

When the input is already the base case (n=0 for factorial, n≤1 for Fibonacci), the function returns immediately — no recursive calls made, constant time O(1).

🔢 factorial(0) — base case, returns 1 instantly
Stack depth: 0
📋 Call Log

Average Case — O(N) Linear Depth

Computing factorial(4) makes N recursive calls before hitting the base case, then unwinds multiplying results back up. Stack depth = N.

🔢 factorial(4) — 4 recursive calls, stack depth = 4
Stack depth: 0
📋 Call Log

Worst Case — O(2ⁿ) Exponential Branching

Fibonacci creates a binary call tree — each call spawns two more calls. fib(5) makes ~2⁵=32 total calls. Space: O(N) stack depth.

🌿 fib(5) — exponential O(2ⁿ) calls, deep branching tree

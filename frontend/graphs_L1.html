Understanding Graph

💡 What is a Graph?

A Graph is a non-linear data structure consisting of vertices and edges. The vertices are sometimes referred to as nodes, and the edges are lines or arcs that connect any two nodes in the graph. More formally, a Graph is composed of a set of vertices V and a set of edges E. The graph is denoted by G(V, E).

🤔 Why Use Graphs?

Graphs model relationships between objects. Unlike trees, graphs can have cycles, multiple paths between nodes, and can represent far more complex real-world structures. They are foundational to network analysis, pathfinding, and scheduling problems.

  • Social Networks — users as vertices, friendships as edges
  • Maps & Navigation — cities as vertices, roads as edges (weighted by distance)
  • Web Crawling — pages as vertices, hyperlinks as edges
  • Dependency Resolution — packages as vertices, dependencies as directed edges
  • Electrical Circuits — components as vertices, wires as edges

🔄 Key Terminology

⚪ Vertex (Node)
A fundamental unit of a graph. Represents an entity — a city, person, web page, etc. A graph with V vertices has indices 0 to V-1.
— Edge
A connection between two vertices. Can be directed (one-way arrow) or undirected (bidirectional). Can also carry a weight.
🔢 Degree
Number of edges incident to a vertex. In directed graphs: in-degree (incoming) and out-degree (outgoing) are counted separately.
🔗 Adjacent
Two vertices are adjacent (neighbours) if there is an edge directly connecting them. The adjacency list stores all neighbours of each vertex.
🔄 Path
A sequence of vertices where each consecutive pair is connected by an edge. A simple path visits each vertex at most once.
🔁 Cycle
A path that starts and ends at the same vertex. A graph with no cycles is called a DAG (Directed Acyclic Graph).

Types of Graphs

1
Undirected Graph
Edges have no direction. If there is an edge (u, v), you can travel both u→v and v→u. Example: friendship network.
2
Directed Graph (Digraph)
Edges have a direction — arrow from source to destination. Edge (u→v) does NOT imply (v→u). Example: Twitter follows.
3
Weighted Graph
Each edge carries a numeric weight (cost, distance, time). Used in shortest path algorithms like Dijkstra's and Bellman-Ford.
4
Connected Graph
Every vertex is reachable from every other vertex. A disconnected graph has isolated components with no path between them.
5
Cyclic / Acyclic
A cyclic graph contains at least one cycle. An acyclic graph has none. Trees are a special case of connected acyclic undirected graphs.
6
DAG
Directed Acyclic Graph — directed with no cycles. Used in scheduling, topological sort, build systems (Makefiles, npm), and neural networks.

Graph Representations

There are two most common ways to represent a graph. The choice affects time and space complexity of graph algorithms significantly.

Feature Adjacency Matrix Adjacency List
Space O(V²) O(V + E)
Check edge (u,v) O(1) O(degree)
Find all neighbours of v O(V) O(degree)
Add edge O(1) O(1)
Best for Dense graphs, quick edge checks Sparse graphs, BFS/DFS traversal
Cache performance Good (2D array) Moderate (linked lists / vectors)
Rule of thumb: Use Adjacency Matrix when V is small and the graph is dense (E ≈ V²). Use Adjacency List when the graph is sparse (E ≪ V²) or you frequently traverse neighbours — which is the common case for BFS and DFS.
🔢

Adjacency Matrix Representation

An adjacency matrix represents a graph as a boolean 2D matrix of 0s and 1s. For a graph with n vertices, create a 2D array adjMat[n][n]:

  • If there is an edge from vertex i to vertex j, mark adjMat[i][j] = 1
  • If there is no edge, mark adjMat[i][j] = 0
  • For undirected graphs, the matrix is symmetric: adjMat[i][j] == adjMat[j][i]

⬜ Undirected Graph — Adjacency Matrix

Symmetric

Consider 3 vertices (0, 1, 2) and edges: (0,1), (0,2), (1,2). Each undirected edge sets two matrix cells.

1
Initialize
Create a 3×3 matrix filled with all 0s — no edges exist yet.
adjMat[3][3] = {0}
2
Edge (0, 1)
Set both mat[0][1]=1 and mat[1][0]=1 since the graph is undirected.
mat[0][1]=1, mat[1][0]=1
3
Edge (0, 2)
Set mat[0][2]=1 and mat[2][0]=1.
mat[0][2]=1, mat[2][0]=1
4
Edge (1, 2)
Set mat[1][2]=1 and mat[2][1]=1. Matrix is now complete.
mat[1][2]=1, mat[2][1]=1

#include <iostream>
#include <vector>
using namespace std;

vector<vector<int>> createGraph(int V, vector<vector<int>> &edges) {
    vector<vector<int>> mat(V, vector<int>(V, 0));

    // Add each edge to the adjacency matrix
    for (auto &it : edges) {
        int u = it[0];
        int v = it[1];
        mat[u][v] = 1;
        mat[v][u] = 1; // undirected — set both directions
    }
    return mat;
}

int main() {
    int V = 3;
    vector<vector<int>> edges = {{0,1},{0,2},{1,2}};
    vector<vector<int>> mat = createGraph(V, edges);

    cout << "Adjacency Matrix:\n";
    for (int i = 0; i < V; i++) {
        for (int j = 0; j < V; j++)
            cout << mat[i][j] << " ";
        cout << endl;
    }
    return 0;
}

public class Graph {
    static int[][] createGraph(int V, int[][] edges) {
        int[][] mat = new int[V][V];

        for (int[] e : edges) {
            mat[e[0]][e[1]] = 1;
            mat[e[1]][e[0]] = 1; // undirected
        }
        return mat;
    }

    public static void main(String[] args) {
        int V = 3;
        int[][] edges = {{0,1},{0,2},{1,2}};
        int[][] mat = createGraph(V, edges);

        System.out.println("Adjacency Matrix:");
        for (int[] row : mat) {
            for (int val : row)
                System.out.print(val + " ");
            System.out.println();
        }
    }
}

Adjacency List Representation

An adjacency list uses an array of lists (or vector of vectors) to store edges. The size of the array equals the number of vertices n. Each index i stores a list of all vertices adjacent to vertex i.

  • adjList[0] → all neighbours of vertex 0
  • adjList[1] → all neighbours of vertex 1, and so on
  • Much more space-efficient for sparse graphs: O(V + E) vs O(V²)

vector<vector<int>> createGraph(int V, vector<vector<int>> &edges) {
    vector<vector<int>> adj(V);

    for (auto &it : edges) {
        int u = it[0], v = it[1];
        adj[u].push_back(v);

    }
    return adj;
}

        cout << i << ": ";
        for (int j : adj[i]) cout << j << " ";

    }
    return 0;
}

Java 
import java.util.*;

public class Graph {
    static List<List<Integer>> createGraph(int V, int[][] edges) {
        List<List<Integer>> adj = new ArrayList<>();
        for (int i = 0; i < V; i++) adj.add(new ArrayList<>());

        for (int[] e : edges) {
            adj.get(e[0]).add(e[1]);
            adj.get(e[1]).add(e[0]); // undirected
        }
        return adj;
    }

    public static void main(String[] args) {
        int[][] edges = {{0,1},{0,2},{1,2}};
        List<List<Integer>> adj = createGraph(3, edges);
        for (int i = 0; i < adj.size(); i++)
            System.out.println(i + ": " + adj.get(i));
    }
}
C 
#include <stdio.h>
#include <stdlib.h>
#define V 3

typedef struct Node {
    int val;
    struct Node* next;
} Node;

Node* adj[V];

void addEdge(int u, int v) {
    Node* n1 = (Node*)malloc(sizeof(Node));
    n1->val = v; n1->next = adj[u]; adj[u] = n1;
    Node* n2 = (Node*)malloc(sizeof(Node));
    n2->val = u; n2->next = adj[v]; adj[v] = n2; // undirected
}

int main() {
    addEdge(0,1); addEdge(0,2); addEdge(1,2);
    for (int i = 0; i < V; i++) {
        printf("%d: ", i);
        for (Node* cur = adj[i]; cur; cur = cur->next)
            printf("%d ", cur->val);
        printf("\n");
    }
    return 0;
}

Real-World Analogies

Graphs are everywhere in the real world. Once you see them, you can't unsee them:

1
Google Maps
Cities are vertices. Roads are weighted edges (distance/time). Dijkstra's algorithm finds the shortest path — a classic graph problem.
2
Social Networks
Users are vertices. Friendship = undirected edge. Follow = directed edge. "Friends of friends" is BFS on the friendship graph.
3
Internet / Web
Web pages are vertices. Hyperlinks are directed edges. Google's PageRank works by analysing this massive directed graph.
4
Package Dependencies
Packages are vertices, dependencies are directed edges. npm/pip use topological sort on a DAG to install in correct order.

What's Next?

Now that you understand graph representations, the next step is learning how to traverse a graph and search for paths:

🔍 BFS — Breadth-First Search
Explores all neighbours at current depth before going deeper. Uses a Queue. Finds the shortest path in unweighted graphs. O(V+E).
📦 DFS — Depth-First Search
Explores as far as possible before backtracking. Uses a Stack (or recursion). Used for cycle detection, topological sort, connected components. O(V+E).
🛣️ Shortest Path
Dijkstra's for non-negative weights. Bellman-Ford handles negative weights. Floyd-Warshall finds all-pairs shortest paths.
🌲 Spanning Tree
Prim's and Kruskal's algorithms find the Minimum Spanning Tree (MST) — a tree connecting all vertices with minimum total edge weight.

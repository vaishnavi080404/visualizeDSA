Understanding Bubble Sort

💡 What is Bubble Sort?

Bubble Sort is a simple comparison-based sorting algorithm. It works by repeatedly stepping through the array, comparing adjacent elements and swapping them if they are in the wrong order. In each pass, the largest unsorted element "bubbles up" to its correct position at the end, just like air bubbles rising to the surface of water.

🤔 Why Use Bubble Sort?

  • Easy to Understand: One of the simplest sorting algorithms — ideal for learning comparison-based sorting.
  • Stable Sort: Equal elements maintain their original relative order after sorting.
  • Adaptive: With an optimized version, it runs in O(N) on an already-sorted array.

Real-World Analogy

Think of bubble sort as repeatedly swapping neighbours until everything settles into the right order — just like these everyday situations:

1
Bubbles Rising in Water
Larger air bubbles rise past smaller ones to reach the top first — just like large values "bubble up" to the end of the array each pass.
2
Sorting People in a Queue by Height
Walk down the line, if a taller person is in front of a shorter one, swap them. After one full walk, the tallest person is at the back.
3
Arranging Books by Thickness
Compare adjacent books — if a thicker one is left of a thinner one, swap them. Repeat from the beginning until no swaps occur.
4
Settling Sediment in a Jar
Shake a jar of mixed particles — heavier ones sink and lighter ones rise through repeated neighbour-swaps until everything settles in order.
⚙️

How it Works & Key Concepts

🔄 The Mechanism

Bubble Sort makes multiple passes through the array. In each pass, it compares every pair of adjacent elements. If the left element is greater than the right, they are swapped. After each full pass, the largest remaining unsorted element is guaranteed to be in its correct position at the end. The process repeats for N-1 passes until the array is sorted.

📚 Basic Terminology

Pass: One full traversal of the array comparing adjacent elements.
Swap: Exchanging two adjacent elements when they are in the wrong order.
Stable Sort: Elements with equal values retain their original relative order.
Swapped Flag: An optimization flag that stops early if no swaps occur in a pass (already sorted).

What Happened?

In each of the N-1 passes, adjacent elements were compared and swapped if out of order. The largest unsorted element "bubbled up" to the end in each pass. Total comparisons: N*(N-1)/2. With the optimized version using a swapped flag, the algorithm stops early if no swaps occur — achieving O(N) on already-sorted arrays.

Implementation

1
Optimized Method (with Swapped Flag)

The optimized approach adds a swapped flag to the standard nested loop. If no swaps occur during a full pass, it means the array is already sorted and the algorithm terminates early — improving best-case performance to O(N).

Time Complexity
O(N²)
Best Case
O(N)
Space Complexity
O(1)

Algorithm:

  • Loop i from 0 to N-1 (outer loop — number of passes).
  • Set a swapped flag to false at the start of each pass.
  • Loop j from 0 to N-i-2 (inner loop — compare adjacent pairs).
  • If arr[j] > arr[j+1], swap them and set swapped = true.
  • If swapped is still false after a full pass, break — array is sorted.
  • Repeat until all passes complete or early exit triggered.

2
Recursive Method

The recursive approach performs one full pass of bubble sort (moving the largest element to the end), then recursively sorts the remaining N-1 elements. The base case is when there is only one element left.

Time Complexity
O(N²)
Space Complexity
O(N)

Algorithm:

  • Base Case: If n == 1, array has one element — already sorted, return.
  • Perform one full pass: compare arr[j] and arr[j+1] for j from 0 to n-2.
  • Swap if arr[j] > arr[j+1]. This places the largest at position n-1.
  • Recursive Step: Call the function with n-1 to sort the remaining subarray.

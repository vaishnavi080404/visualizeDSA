Understanding Linear Search

💡 What is Linear Search?

Linear search is a searching algorithm where the data set is traversed from one end to the other until the desired value is found. It is also known as Sequential Search because it checks every element in sequence.

🤔 Why Use Linear Search?

  • No Sorting Required: Unlike Binary Search, it works on unsorted lists.
  • Simple Implementation: Very easy to write and understand for beginners.
  • Small Datasets: Efficient enough for lists with a few hundred elements.
Time Complexity (Worst)
O(N)
Space Complexity
O(1)

How it Works & Key Concepts

🔄 The Mechanism

Linear search works by sequentially checking each element in the list until the desired value is found or the end of the list is reached. If the element is found, the algorithm returns its index. If the loop finishes without a match, it returns -1.

📚 Basic Terminology

Target: The specific value you are searching for.
Traversal: The process of visiting each element in order.
Sentinel: A special value (like -1) used to signal "Not Found".

Implementation

1
Iterative Method (Standard Loop)

The iterative approach uses a simple loop to traverse the array from index 0 to N-1. In each iteration, it compares the current element with the target value.

Time Complexity
O(N)
Space Complexity
O(1)

Algorithm:

  • Start from the first element (index 0).
  • Compare the target value with the current element of the array.
  • If the target matches, return the current index.
  • If the target does not match, move to the next element.
  • Repeat until the element is found or the end of the array is reached.
  • If the end is reached without a match, return -1.
C++
2
Recursive Method

The recursive approach breaks the problem into smaller sub-problems. It checks the current element and then calls itself to check the rest of the array.

Time Complexity
O(N)
Space Complexity
O(N)

Algorithm:

  • Base Case 1: If index is out of bounds, return -1 (Not Found).
  • Base Case 2: If element at current index == target, return index (Found).
  • Recursive Step: Return the result of the search function called with index + 1.

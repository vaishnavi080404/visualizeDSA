Understanding the core concepts of the Heap data structure and its operations

📚

Understanding Heap

💡 What is a Heap?

A Heap is a specialized complete binary tree data structure that satisfies the Heap Property. In a Max-Heap, every parent node has a value greater than or equal to its children. In a Min-Heap, every parent node has a value less than or equal to its children. This property ensures that the root is always the maximum (Max-Heap) or minimum (Min-Heap) element in the tree.

Heaps are not a sorted structure — only the parent-child relationship is enforced, not any sibling relationship. This makes heaps extremely efficient for priority-based operations.

🤔 Why Use a Heap?

Heaps are primarily used to implement Priority Queues, where the highest (or lowest) priority element needs to be served first. The key advantage is that the maximum or minimum element is always accessible in O(1) time, while insert and delete still take only O(log N) time.

  • Priority Queues — task scheduling, event-driven simulations
  • Heap Sort — an O(N log N) in-place sorting algorithm
  • Graph Algorithms — Dijkstra's shortest path, Prim's MST use min-heaps
  • K-th Largest/Smallest — efficiently find K-th element in a stream
  • Merge K Sorted Lists — classic interview problem solved with min-heaps
  • Job Scheduling — OS schedulers use priority-heap based queues

🔄 How Does a Heap Work?

A heap is stored as a compact array (no pointers needed). For a node at index i:

  • Left child is at index 2*i + 1
  • Right child is at index 2*i + 2
  • Parent is at index (i - 1) / 2

When an element is inserted, it is added at the end of the array and then "heapified up" (bubbled up) to restore the heap property. When the root is removed, the last element replaces it and is "heapified down" (sifted down).

The Heap Property

Every heap must satisfy two fundamental properties simultaneously:

1️⃣ Complete Binary Tree Property

A heap must be a complete binary tree — all levels are fully filled except possibly the last level, which is filled from left to right. This ensures the array representation is compact with no gaps.

2️⃣ Heap Ordering Property

For every node i in the tree:

  • Max-Heap: arr[parent(i)] ≥ arr[i] — every parent is greater than or equal to its children
  • Min-Heap: arr[parent(i)] ≤ arr[i] — every parent is less than or equal to its children
Note: There is no ordering between siblings in a heap. The left child may be larger or smaller than the right child — only the parent-child relationship is enforced.

Key Concepts

Heapify Up (Bubble Up)
After inserting at the end, compare the new element with its parent. If it violates the heap property, swap and continue up. Runs in O(log N).
Heapify Down (Sift Down)
After removing the root, replace it with the last element and compare with children. Swap with the appropriate child and continue down. O(log N).
Build Heap
Convert an arbitrary array into a valid heap by calling heapify-down on all non-leaf nodes from bottom to top. Surprisingly O(N), not O(N log N).
Priority Queue
The most common use of heaps. Each element has a priority; the highest-priority element is always served first. Implemented using a heap internally.
Heap Height
A heap with N elements has height ⌊log₂N⌋. This bounded height is why all operations are O(log N) — you never traverse more than the height.
Leaf Nodes
In a 0-indexed heap of N elements, all nodes from index ⌊N/2⌋ to N-1 are leaf nodes. Leaves have no children and are always valid heap nodes.
🔧

Core Operations — Step by Step

⬆ Insert & Heapify Up

To insert into a Max-Heap: add the element at the end, then compare with its parent and swap upward until the heap property is restored.

⬇ Delete Root & Heapify Down

To delete the root from a Max-Heap: replace it with the last element, remove the last position, then heapify down.

🔧

Types of Heaps

1. Binary Heap

The most common form. A complete binary tree where each node has at most two children. Stored compactly as an array. All standard heap operations are O(log N). This is what heapq in Python and priority_queue in C++ implement internally.

2. Binomial Heap

A arr of binomial trees. Supports efficient merge of two heaps in O(log N). More complex to implement but useful when merging heaps is frequent. Insert: O(log N), Delete-min: O(log N), Merge: O(log N).

3. Fibonacci Heap

Achieves amortized O(1) for insert and decrease-key operations, making it theoretically optimal for graph algorithms like Dijkstra's and Prim's. However, the constant factors are large, so binary heaps are preferred in practice.

4. d-ary Heap (k-ary Heap)

A generalization of binary heaps where each node has d children instead of 2. Higher branching factor means shorter height (O(log_d N)) but more comparisons per heapify-down step. A 4-ary heap often performs better in practice due to cache efficiency.

🌍

Real-World Analogies

Heaps model many real-world systems where the most important item must always be accessible first:

1
Hospital Emergency Room
Patients are treated based on severity (priority), not arrival order. The most critical patient is always seen first — a Max-Heap by severity score.
2
Job Scheduler (OS)
The operating system uses a Min-Heap to pick the process with the shortest remaining time or highest priority to run next on the CPU.
3
Stock Market Orders
A stock exchange uses two heaps — a Max-Heap for buy orders (highest bid first) and a Min-Heap for sell orders (lowest ask first) — to match trades.
4
Navigation (Dijkstra)
GPS apps find the shortest path using Dijkstra's algorithm with a Min-Heap — always explore the closest unvisited node next.
📝

Basic Terminologies

Root
The top element of the heap (index 0). Always holds the maximum value in a Max-Heap or minimum value in a Min-Heap.
Heapify
The process of rearranging elements to maintain the heap property after an insertion or deletion. Can go up (insert) or down (delete).
Heap Size
The current number of elements stored in the heap. Used to identify the last node and boundary for heap operations.
Overflow
Condition when trying to insert into a fixed-size heap that is already full. Dynamic heaps (linked-list or resizing array) never overflow.
Underflow
Condition when trying to delete or peek from an empty heap. Always check isEmpty before performing delete or peek operations.
Complete Binary Tree
A binary tree where all levels are filled except possibly the last, which is filled left to right. Guarantees O(log N) height for all operations.

Insert (Push) Operation
▼

Insert element at the end, then heapify up by comparing with parent and swapping if needed.

Algorithm:

  • Add the new element at the end (heap size position)
  • Compare with parent: index = (i-1)/2
  • If element > parent (Max-Heap), swap them
  • Repeat from new position until heap property is satisfied

C++
void insert(int val) {
    // Add new element at the end
    heap.push_back(val);
    int i = heap.size() - 1;

    // Heapify up — compare with parent
    while (i > 0) {
        int parent = (i - 1) / 2;
        if (heap[i] > heap[parent]) {  // Max-Heap
            swap(heap[i], heap[parent]);
            i = parent;
        } else break;  // Heap property satisfied
    }
}

Delete Max (Extract Root)
▼

Remove the root (maximum element). Replace with last element, then heapify down.

C++
int extractMax() {
    if (heap.empty()) return -1;
    int maxVal = heap[0];        // Store max
    heap[0] = heap.back();       // Move last to root
    heap.pop_back();             // Remove last
    heapifyDown(0);             // Restore property
    return maxVal;
}

void heapifyDown(int i) {
    int n = heap.size();
    while (true) {
        int largest = i;
        int left  = 2*i + 1;
        int right = 2*i + 2;
        if (left  < n && heap[left]  > heap[largest]) largest = left;
        if (right < n && heap[right] > heap[largest]) largest = right;
        if (largest == i) break;
        swap(heap[i], heap[largest]);
        i = largest;
    }
}
Python
def extract_max(self):
    if not self.heap:
        return -1
    max_val = self.heap[0]
    self.heap[0] = self.heap[-1]
    self.heap.pop()
    self.heapify_down(0)
    return max_val

def heapify_down(self, i):
    n = len(self.heap)
    while True:
        largest, left, right = i, 2*i+1, 2*i+2
        if left  < n and self.heap[left]  > self.heap[largest]:
            largest = left
        if right < n and self.heap[right] > self.heap[largest]:
            largest = right
        if largest == i: break
        self.heap[i], self.heap[largest] = \
            self.heap[largest], self.heap[i]
        i = largest

Build Heap from Array — O(N)
▼

Convert any array into a valid heap in O(N) time by calling heapify-down on all internal (non-leaf) nodes from the bottom up. Start from index N/2 - 1 down to 0.

Time
O(N)
vs N inserts
O(N log N)
Why O(N)? Most nodes are near the bottom of the tree (leaves), so they need very little heapification work. The mathematical summation of all work converges to O(N) rather than O(N log N).

Java

C++
// Build Max-Heap from arbitrary array — O(N)
void buildHeap(vector<int>& arr) {
    heap = arr;
    int n = heap.size();
    // Start from last non-leaf, go up to root
    for (int i = n/2 - 1; i >= 0; i--)
        heapifyDown(i);
}

// Usage
vector<int> arr = {3, 1, 6, 5, 2, 4};
buildHeap(arr);
// Result: [6, 5, 4, 1, 2, 3] — valid Max-Heap

C — No built-in, manual array heap
#include <stdio.h>
#define MAX 100

int heap[MAX], size = 0;

void swap(int *a, int *b) { int t=*a; *a=*b; *b=t; }

void insert(int v) {
    heap[size++] = v;
    int i = size - 1;
    while (i > 0 && heap[(i-1)/2] < heap[i]) {
        swap(&heap[i], &heap[(i-1)/2]);
        i = (i-1)/2;
    }
}

int extractMax() {
    if (!size) return -1;
    int max = heap[0];
    heap[0] = heap[--size];
    int i = 0;
    while (true) {
        int l=2*i+1, r=2*i+2, lg=i;
        if (lheap[lg]) lg=l;
        if (rheap[lg]) lg=r;
        if (lg==i) break;
        swap(&heap[i], &heap[lg]); i=lg;
    }
    return max;
}

Applications of Heap

🏆 Heap Sort

Build a Max-Heap from the array (O(N)), then repeatedly extract the maximum and place it at the end of the array. This gives a sorted array in ascending order. Time: O(N log N), Space: O(1) in-place.

📊 K-th Largest / Smallest Element

To find the K-th largest element in a stream: maintain a Min-Heap of size K. For each new element, push it and pop if size exceeds K. The root is always the K-th largest. Time: O(N log K) — far better than sorting all N elements.

🔀 Merge K Sorted Arrays

Use a Min-Heap with one element from each of the K arrays. Extract the minimum, add to result, then push the next element from the same array. Time: O(N log K) where N is total elements.

🗺️ Dijkstra's Shortest Path

Use a Min-Heap (priority queue) to always explore the closest unvisited node first. This reduces Dijkstra's complexity from O(V²) to O((V + E) log V).

📱 Median in a Data Stream

Maintain two heaps: a Max-Heap for the lower half and a Min-Heap for the upper half. The median is either the root of one or the average of both roots. All operations in O(log N).

⚠️

When NOT to Use a Heap

🔍 Need Full Sorted Order
Heaps only guarantee the extreme element at the root. For fully sorted access, use a sorting algorithm or balanced BST.
🔎 Random Access by Key
You can't efficiently look up an arbitrary element by key in a heap. Use a hash map or BST for key-based lookups.
📊 Range Queries
Finding all elements in range [lo, hi] is O(N) in a heap. Use a BST or segment tree for efficient range queries.
🔄 Frequent Decrease-Key
Binary heaps don't support decrease-key efficiently (O(N) to find the element). Use a Fibonacci Heap for this use case.
Rule of thumb: Use a Heap when you need repeated access to the extreme element (max or min) with O(1) peek and O(log N) insert/delete. If you need full sorted order, range queries, or arbitrary key access — choose a BST or hash map.

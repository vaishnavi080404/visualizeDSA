Understanding Linked List

💡 What is a Linked List?

A Linked List is a linear data structure where elements are stored in nodes, and each node holds a value plus a pointer (link) to the next node. Unlike arrays, nodes are not stored at contiguous memory locations — each node can sit anywhere in memory and is connected to the next via a reference.

🤔 Why Use a Linked List?

Arrays have a fixed size and expensive insertion/deletion (requires shifting elements). Linked lists solve this — insertion and deletion at any position costs O(1) if you already have the pointer to that node. They're the backbone of stacks, queues, deques, graphs (adjacency lists), and more.

🔄 How Does it Work?

Every linked list has a head pointer that points to the first node. Each node has two parts: data (the value) and next (a pointer to the next node). The last node's next points to NULL, marking the end of the list. You traverse the list by following these links from head to NULL.

Real-World Analogy

Think of a treasure hunt — each clue tells you where to find the next one. You start from the first clue (head) and follow pointers until there's no next clue (NULL).

1
Train Compartments
Each compartment is connected to the next one. You board from the engine (head) and walk forward through couplings (pointers).
2
Treasure Hunt Clues
Each clue (node) has data (the riddle) and a link (next location). You follow from clue to clue until no next pointer exists.
3
Browser History
Each visited page links to the previous one. Going back follows the prev pointer — exactly how a doubly linked list works.
4
Music Playlist
Each song knows the next song to play. Skip ahead = follow next; shuffle = relink pointers. A circular playlist loops back to head.

Key Concepts

Node
The basic building block. Contains data and a reference (pointer) to the next node in the sequence.
Head
Pointer to the first node in the list. This is your entry point — without head, you can't access anything.
Tail
The last node in the list. Its next pointer is NULL (or back to head for circular lists).
Traversal
Walking from head to tail via next pointers. O(n) operation — you must visit each node in sequence.
Insertion
O(1) at head or tail (with tail pointer). O(n) at arbitrary position since you must find the node first.
Deletion
Redirect the previous node's next pointer around the deleted node. Free the memory in languages like C/C++.
🔧

Types of Linked List

➡️ Singly Linked List

Most Common

Each node has one pointer — next. You can only traverse forward (head → tail). Simple, memory-efficient.

↔️ Doubly Linked List

Bidirectional

Each node has two pointers: next and prev. You can traverse both forward and backward. Uses more memory but enables O(1) deletion when you have a pointer to any node.

🔄 Circular Linked List

No NULL End

The tail node's next points back to the head instead of NULL. Useful for round-robin scheduling, circular buffers, and playlists. Can be singly or doubly circular.

⚖️

Linked List vs Array

Knowing when to pick one over the other is what matters in practice.

Property Linked List Array
Memory Layout Non-contiguous Contiguous
Access by Index O(n) — traverse O(1) — direct
Insert at Head O(1) O(n) — shift all
Delete at Middle O(1)* O(n) — shift
Memory Usage Extra per node (pointer) Just the data
Size Dynamic Fixed (static)
Cache Friendly No Yes

*O(1) deletion only if you already hold the pointer. Finding the node costs O(n).

📝

Basic Terminologies

Node
Fundamental unit containing data and a pointer (next) to the following node.
Head
Reference to the first node. Losing head means losing access to the whole list.
Tail
Last node in the list. Its next pointer is NULL (or head for circular lists).
Next Pointer
Stores the memory address of the next node. This is what makes the list "linked".
Prev Pointer
Only in doubly linked lists. Points to the previous node — enables backward traversal.
NULL Terminator
A next pointer set to NULL indicates the end of the list. Always check before accessing.

Insert at Head
▼

Adds a new node at the beginning of the list. The trick here is: point the new node to the current head, then update head to the new node.

Time Complexity
O(1)
Space Complexity
O(1)

Algorithm:

  • Create a new node with the given value
  • Set newNode.next = head
  • Update head = newNode

C 
void insertAtHead(struct Node** head, int val) {
    struct Node* node = newNode(val);
    
    // New node points to current head
    node->next = *head;
    
    // head is now the new node
    *head = node;
}

Insert at Tail
▼

Walk to the last node and link the new node there. O(n) unless you maintain a tail pointer.

Time Complexity
O(n)
With Tail Ptr
O(1)

Algorithm:

  • If head is NULL, the new node becomes head
  • Traverse using a ptr until ptr.next == NULL
  • Set ptr.next = newNode
C
C++
Java
Python
JavaScript
C 
void insertAtTail(struct Node** head, int val) {
    struct Node* node = newNode(val);
    
    // Empty list — new node is head
    if (*head == NULL) { *head = node; return; }
    
    struct Node* ptr = *head;
    while (ptr->next != NULL) ptr = ptr->next;
    ptr->next = node;
}
C++ 
void insertAtTail(Node*& head, int val) {
    Node* node = new Node(val);
    if (!head) { head = node; return; }
    
    Node* ptr = head;
    while (ptr->next) ptr = ptr->next;
    ptr->next = node;
}
Java 
public void insertAtTail(int val) {
    Node node = new Node(val);
    if (head == null) { head = node; return; }
    
    Node ptr = head;
    while (ptr.next != null) ptr = ptr.next;
    ptr.next = node;
}
Python 
def insert_at_tail(self, val):
    node = Node(val)
    if not self.head:
        self.head = node
        return
    ptr = self.head
    while ptr.next:
        ptr = ptr.next
    ptr.next = node
JavaScript 
insertAtTail(val) {
    const node = new Node(val);
    if (!this.head) { this.head = node; return; }
    
    let ptr = this.head;
    while (ptr.next) ptr = ptr.next;
    ptr.next = node;
}

3
Delete a Node
▼

Find the node to delete, then redirect its predecessor's next pointer to skip it. Always handle the edge case where it's the head.

Time Complexity
O(n)
Space Complexity
O(1)

Algorithm:

  • If head.data == key, move head = head.next and free old head
  • Traverse using ptr until ptr.next.data == key
  • Set ptr.next = ptr.next.next
  • Free the deleted node (C/C++)
C
C++
Java
Python
JavaScript
C 
void deleteNode(struct Node** head, int key) {
    struct Node* ptr = *head;
    
    // Edge case: deleting head
    if (ptr && ptr->data == key) {
        *head = ptr->next;
        free(ptr);
        return;
    }
    
    while (ptr->next && ptr->next->data != key)
        ptr = ptr->next;
    
    if (!ptr->next) return; // key not found
    
    struct Node* tmp = ptr->next;
    ptr->next = tmp->next;
    free(tmp);
}
C++ 
void deleteNode(Node*& head, int key) {
    if (head && head->data == key) {
        Node* tmp = head;
        head = head->next;
        delete tmp;
        return;
    }
    Node* ptr = head;
    while (ptr->next && ptr->next->data != key)
        ptr = ptr->next;
    
    if (!ptr->next) return;
    Node* tmp = ptr->next;
    ptr->next = tmp->next;
    delete tmp;
}
Java 
public void deleteNode(int key) {
    if (head != null && head.data == key) {
        head = head.next;
        return;
    }
    Node ptr = head;
    while (ptr.next != null && ptr.next.data != key)
        ptr = ptr.next;
    
    if (ptr.next != null)
        ptr.next = ptr.next.next; // GC handles cleanup
}
Python 
def delete_node(self, key):
    if self.head and self.head.data == key:
        self.head = self.head.next
        return
    ptr = self.head
    while ptr.next and ptr.next.data != key:
        ptr = ptr.next
    if ptr.next:
        ptr.next = ptr.next.next
JavaScript 
deleteNode(key) {
    if (this.head?.data === key) {
        this.head = this.head.next;
        return;
    }
    let ptr = this.head;
    while (ptr?.next && ptr.next.data !== key)
        ptr = ptr.next;
    
    if (ptr?.next)
        ptr.next = ptr.next.next;
}

void printList(struct Node* head) {
    struct Node* ptr = head;
    while (ptr != NULL) {
        printf("%d → ", ptr->data);
        ptr = ptr->next;
    }
    printf("NULL\n");
}

void printList(Node* head) {
    for (Node* ptr = head; ptr; ptr = ptr->next)

}

public void printList() {
    Node ptr = head;

        System.out.print(ptr.data + " → ");

}

def print_list(self):

printList() {
